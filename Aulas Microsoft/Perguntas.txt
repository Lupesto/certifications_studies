Identificar tabelas e colunas no Dataverse

Tipos de tabelas
Padrão, gerenciado e personalizado


Padrão
 tabelas prontas para uso, 
Contas, unidade de negócios, contato, tarefa e tabelas de usuário são exemplos de tabelas padrão no Dataverse. 
A maioria podem ser personalizadas. 
Tabelas importadas como parte de uma solução gerenciada e definidas como personalizáveis também aparecem como tabelas padrão. Qualquer usuário com privilégios adequados pode personalizar essas tabelas desde que o personalizável seja true. 




Gerenciado
não são personalizáveis
foram importadas para o ambiente como parte de uma solução gerenciada.


Personalizado 
As tabelas personalizadas são tabelas não gerenciadas importadas de uma solução não gerenciada ou tabelas criadas diretamente no ambiente do Dataverse. 
Qualquer usuário com privilégios adequados pode personalizar totalmente essas tabelas.

Colunas
forma de armazenar uma informação discreta dentro de uma linha em uma tabela. 

como uma coluna no Excel. 

você pode armazenar dados de um determinado tipo em uma coluna que corresponda ao tipo específico. 
Por exemplo, em soluções que exigem valores de datas, armazene a data em uma coluna com o tipo Data. Da mesma forma, para armazenar um número, faça isso em uma coluna com o tipo Número.

acima de cem colunas considere outras alternativas



Cada banco de dados no Microsoft Dataverse começa com um conjunto padrão de tabelas, e cada tabela padrão tem um conjunto padrão de colunas.

Noções básicas sobre relações


Para criar uma solução eficiente e escalonável
dividir os dados em diferentes contêineres (tabelas). Tentar armazenar tudo em um único contêiner provavelmente não seria eficiente e dificultaria o trabalho e a compreensão.
crie mais de uma tabela com suas categorias, para tornar o negócio compreensivel
facilita a criação de relatórios e alto desempenho




Tabelas que se relacionam entre si têm uma conexão relacional. 


Há vários formatos de relações entre tabelas, mas os dois mais comuns são: um-para-muitos e muitos para muitos, ambos com suporte do Microsoft Dataverse.


um-para-muitos (pai-filho).
pode ter zero, um ou muitos itens de linha (linhas filho), mas o item de linha sempre estará relacionado a apenas uma fatura (linha pai). Normalmente, as linhas filhas não existem sem uma linha pai.
coluna com valores exclusivos identificam a linha pai
Essa coluna exclusiva é chamada de chave.
O mesmo valor (a chave pai) é armazenado nas linhas filho relacionadas. Essa coluna é chamada de chave estrangeira quando a linha filho é usada para armazenar o valor da chave pai. A filtragem é usada para exibir linhas filho com um valor na chave estrangeira que corresponde ao valor de chave na linha pai. Isso permite que os aplicativos exibam as linhas filhas (itens de linha no exemplo anterior) que pertencem a uma linha pai específica (fatura no exemplo anterior). Esse conceito é a base de muitos aplicativos de software de negócios.


dividir as informações em tabelas pode ser dificil mas a microsoft já possui o padrão de tabelas que servem em vários negócios, é mais comprovado e escalonável utilizar o padrão

Regras de negócio

As regras de negócio permitem que você aplique e mantenha a lógica de negócios na camada de dados em vez da camada de aplicativo. 
elas entrarão em vigor independentemente do local no qual você interage com os dados.


podem ser usados para validar dados armazenados ou mostrar mensagens de erro. 


Os aplicativos orientados por modelos podem usar regras de negócios para 
mostrar, ocultar, habilitar ou desabilitar colunas e criar recomendações com base em business intelligence.


maneira eficiente de impor regras, definir valores ou validar dados, independentemente do formulário usado para dados de entrada. 


aumentar a precisão dos dados, simplificar o desenvolvimento de aplicativos e simplificar os formulários apresentados aos usuários finais.



As seguintes ações de regras de negócios podem ser usadas por aplicativos baseados em tela e modelos:
Definir valores de coluna
Limpar valores de coluna
Validar dados e mostrar mensagens de erro


Os aplicativos baseados em modelos também podem usar regras de negócio para:
Mostrar ou ocultar campos (somente aplicativos baseados em modelos)
Habilitar ou desabilitar colunas (somente aplicativos baseados em modelos)
Criar recomendações de negócios baseadas em business intelligence (somente aplicativos baseados em modelos)

 melhor controle dos seus dados. 

Isso pode garantir que a lógica de negócios seja seguida, independentemente de ela estar sendo acessada diretamente do Power Apps, do Power Automate ou até mesmo por meio de uma API. A 


regra é vinculada aos dados, não ao aplicativo.

RESPOSTAS:
Um de seus colegas quer configurar um fluxo para gravar um item em uma lista do SharePoint ao receber um email do chefe. Qual dos itens a seguir o ajudaria a iniciar?
Verificar se há modelos disponíveis para esse cenário. Se houver, utilizá-los; caso contrário, criar um fluxo em branco com um gatilho "Quando um email for recebido".

Os fluxos instantâneos exigem a entrada do usuário, seja pelo Power Apps, por um botão de fluxo ou de outra forma. Não há nenhuma ação para "Quando um email for recebido" quando você está criando um fluxo instantâneo.


Seu departamento deseja configurar um sistema de aprovação para itens que estão sendo gravados em uma lista do SharePoint. Você recebeu a tarefa de criar o sistema. Usando o Flow, qual é a melhor abordagem?
Criar um fluxo de nuvem automatizado do zero e adicionar um gatilho para "Quando um novo item for criado" no SharePoint e ações para as aprovações.

Embora você certamente possa abordar o problema dessa forma, é mais fácil usar os fluxos internos Aprovações do SharePoint, pois eles foram projetados para resolver esse problema. Se fosse uma fonte de dados diferente, essa poderia ser a melhor resposta.

 Sua organização começou a usar vários fluxos de aprovação, e os usuários estão reclamando que têm muitas aprovações para acompanhar. Eles não sabem o que foi aprovado e o que ainda está com aprovação pendente. Qual é a maneira mais fácil para eles verificarem?
Pedir para os usuários navegarem até o Centro de Aprovações no Power Automate.

No Centro de Aprovações, os usuários poderão ver as aprovações pendentes e o histórico de aprovações em uma plataforma muito mais fácil de ler. Todas as aprovações que exigem a interação do usuário estarão disponíveis por meio do Centro de Aprovações.


Um de seus colegas deseja postar no Twitter informações sobre itens do SharePoint que foram aprovados. Isso é possível? Em caso afirmativo, como ele faria?
Sim, é possível. Ele pode criar um fluxo para monitorar os itens aprovados na lista do SharePoint e depois tweetar os detalhes.
Isso foi abordado neste módulo; portanto, basta rever o módulo para saber como realizar esse procedimento.

Segurança e administração

 o Power BI se baseia no Azure. plataforma e a infraestrutura de computação em nuvem da Microsoft, que garante o mesmo nível de segurança para Power BI como em outros serviços da Microsoft. Os usuários entram com suas credenciais mantidas no Azure Active Directory e controlam o nível de compartilhamento para todos os relatórios, dados ou dashboards, determinando se os destinatários podem editar ou apenas exibir itens.

 os usuários são responsáveis pelos dados que compartilham. 


A administração do Power BI é o gerenciamento de um locatário do Power BI, incluindo a configuração de políticas de governança, monitoramento de uso, provisionamento de licenças, capacidades e recursos organizacionais.

 O administrador é o guardião de dados, processos e políticas no locatário Power BI. Um administrador do Power BI é um membro essencial de uma equipe que inclui os desenvolvedores de BI, analistas e outras funções.

Componentes do Power Virtual Agents

Os tópicos são roteiros de conversa discretos que, quando usados em conjunto em um único chatbot, permitem que os usuários tenham uma conversa com um chatbot que parece natural e que fui de forma apropriada. 

Também é fácil testar o chatbot sem precisar implantá-lo totalmente sempre que você faz uma pequena alteração. 
um tópico define como uma conversa de chatbot é reproduzida. 
É possível criar tópicos personalizando os modelos fornecidos, criar novos tópicos do zero ou obter sugestões de sites


Um tópico tem frases de gatilho - são frases, palavras-chave ou perguntas que provavelmente um usuário digite relacionadas a um problema específico - e nós de conversa - são os que você usa para definir como um chatbot deve responder e o que ele deve fazer.

A IA usa o reconhecimento vocal natural para analisar o que um cliente realmente digita e encontrar o nó ou a frase de gatilho mais apropriado.





Usar tópicos de amostra e do sistema
Quando você criar um chatbot, vários tópicos serão criados automaticamente para você.

São eles:
tópicos prontos para te mostrar como funciona e tópicos de sistema (n da para excluir)



Entidades
 reconhecimento vocal natural,  capacidade da IA de compreender a intenção de um usuário. 
Por exemplo, o reconhecimento vocal natural é envolvido quando um usuário pode dizer "Eu tentei usar meu vale-presente, mas ele não funciona" e o chatbot é capaz de encaminhar o usuário para o tópico relacionado aos vales-presentes que não estão funcionando - mesmo que a frase exata não esteja listada como uma frase de gatilho.





Um aspecto fundamental do reconhecimento vocal natural é identificar as entidades em uma caixa de diálogo do usuário. Uma entidade pode ser exibida como uma unidade de informação que representa um determinado tipo de assunto real, como um número de telefone, CEP, cidade ou mesmo o nome de uma pessoa.

Entidades predefinidas
 entidades predefinidas,
 informações usadas com mais frequência em caixas de diálogo reais, como idade, cores, números e nomes.

Com o conhecimento concedido por entidades, um chatbot pode reconhecer de forma inteligente as informações relevantes a partir de uma entrada do usuário e salvá-las para uso posterior.



Entidades personalizadas
em algumas ocasiões você precisará ensinar o modelo de reconhecimento vocal do chatbot um conhecimento específico do domínio.



Ações
Você pode habilitar o chatbot para executar uma ação chamando apenas um fluxo do Microsoft Power Automate. 


Publicação
 você pode publicar chatbots para interagir com seus clientes em várias plataformas ou canais. Isso inclui sites ativos, aplicativos móveis e plataformas de mensagens como Microsoft Teams e Facebook.
